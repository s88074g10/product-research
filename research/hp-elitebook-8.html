HP Wolf Security for Business — 使用者操作手冊

每個功能含「怎麼開／何時觸發／事件處理」

Wolf 不是單一程式，而是 8 個獨立子功能組成的套件。下面把每個子功能的「定位 / 怎麼啟用 / 出事怎麼處理」拆開說明，業務遇到客戶問操作問題可以直接照講。

01 HP Sure Click — Email 附件 / 網頁 Micro-VM 隔離

背景：惡意軟體常隱藏在看似清白的檔案中、從網路下載到電腦上、或以 Email 附件形式寄出。
HP 的解法：Sure Click 會以「微型虛擬機器」(Micro-VM) 獨立虛擬容器打開不受信任的網站與檔案。這些 Micro-VM 誘使惡意軟體以為自己正在你的電腦上運行，但只要關閉分頁或檔案，惡意軟體就消失無蹤。
怎麼啟用

預設開啟。開始選單搜尋「Sure Click Secure Browser」可看狀態。Outlook 開附件時自動觸發。

何時觸發

使用者點開不信任來源的 Word / Excel / PDF / 網頁時，左上角會出現「Protected by Sure Click」提示。

事件後處理

關閉檔案 / 分頁 → 沙盒自動銷毀。若 Sure Click 跳警告「偵測到惡意行為」→ 直接關閉視窗，無須額外清理。

02 HP Sure Sense — AI 行為防毒（深度學習 + 神經網絡）

背景：每日有數以千計的惡意軟體變異體誕生，傳統防毒軟體靠特徵檔比對，無法時刻偵測新攻擊。
HP 的解法：Sure Sense 採用專利深度學習算法和先進神經網絡技術，能本能地識別惡意軟體的「行為特徵」，抵禦無法預測的攻擊（包含尚未被命名的零時差攻擊）。
怎麼啟用

預設開啟，常駐背景。可在「HP Wolf Security 主控台」查看防護狀態。

何時觸發

下載 / 執行任何檔案時即時掃描。發現可疑行為（加密大量檔案、改 registry）立刻阻擋。

事件後處理

跳出阻擋通知 → 點「查看詳情」可看哪個程式、哪個動作被擋。誤判可手動加入白名單。

03 HP Sure Start — 全球首款自我修復 BIOS

背景：如果惡意軟體入侵 BIOS，攻擊者就能對你的電腦取得無限制控制權 —— 竊取資料、植入勒索軟體、甚至讓電腦無法操作。由於防毒軟體無法監察韌體攻擊，隱藏在 BIOS 中的惡意軟體幾乎無法偵測。
HP 的解法：Sure Start 是全球首款兼唯一一款懂得自我修復的 BIOS。它不是嘗試找出已知病毒，而是自動識別任何未經授權的 BIOS 變更—— 換言之，連世人前所未見的攻擊都能擋下。
怎麼啟用

BIOS 層級內建，無需設定，永遠 On。開機時若 BIOS 異常會看到「HP Sure Start - Recovering BIOS」訊息。

何時觸發

每次開機自動驗證 BIOS 完整性。被竄改 / 損毀 → 進入修復模式（約 30 秒）。

事件後處理

修復完成自動正常開機。若反覆觸發 → 送修，可能是硬體或主機板異常。

04 HP Sure Admin — 手機授權取代 BIOS 密碼

背景：傳統 BIOS 密碼有兩大痛點 —— 員工離職忘記交接、密碼以明碼存在 IT 工單裡容易外流；要嘛被遺忘讓筆電變磚，要嘛被前員工濫用。
HP 的解法：Sure Admin 把 BIOS 認證改成「螢幕顯示 QR Code → 管理員手機 App 掃描授權」。授權記錄在雲端，員工離職一鍵撤銷，舊手機立刻失效。等於 BIOS 也走「Pin / OAuth」式的現代驗證。
怎麼啟用

需 IT 管理員先設定（搭配 HP Connect 部署）。家用 / 個人客戶可選擇不啟用，繼續用密碼模式。

何時觸發

使用者按 F10 進 BIOS → 螢幕顯示 QR Code → 管理員用手機 App 掃描授權。

事件後處理

離職員工 → IT 在 HP Connect 後台撤銷該員手機授權即可，舊手機無法再進入任何公司 BIOS。

05 HP Sure Recover — 雲端 OS 自動重灌

背景：重新安裝作業系統是消除惡意軟體的有效法門，但傳統重新成像（恢復電腦至原廠設定）相當耗時，要找 USB 隨身碟、燒映像檔、設 BIOS 開機順序、等漫長安裝。中小企業沒 IT 的話得整台送修。
HP 的解法：Sure Recover 只需連網路，作業系統就快速、安全、自動復原。減少停機時間，迅速重新投入工作。連 IT 都不用上門。
怎麼啟用

預設啟用。執行：開機時按 F11（HP Recovery 鍵）→ 選 Sure Recover → 連網即可。

何時觸發

系統壞了開不了機、中勒索病毒、系統檔案損毀。

事件後處理

需要連有線網路（Wi-Fi 也可但建議有線穩定）。整個還原約 30–60 分鐘。資料會清空，務必先備份。

06 HP Sure Run — 防護程序自我守護

背景：惡意軟體經常以「攻擊中的主要防禦手段」為目標 —— 想盡辦法把防毒、防火牆、Wolf 服務關掉或停用，這樣它就能在無人看管下取得電腦更大的存取權限。
HP 的解法：Sure Run 持續監測這些防禦系統，一旦偵測任何變更立即通知使用者；防禦系統被停掉時自動重啟。等於替你的資安防線「裝上自動回血機制」。
怎麼啟用

預設開啟，背景執行不可手動關閉（避免誤關）。

何時觸發

防毒、Sure Sense、Sure Click 等被惡意關閉時，自動重啟服務。

事件後處理

使用者通常無感。資訊主管可在 HP Wolf Controller 看到「服務被外力中止 → 已自動恢復」紀錄。

07 HP Tamper Lock — 機殼開啟偵測 ⚠️

背景：對裝置發動的實體攻擊日益增加：拆機替換快閃記憶裝置、探測 TPM 晶片 取得加密金鑰、發動直接記憶體存取（DMA）攻擊。這些手法純軟體防護完全擋不住，因為攻擊者手上有實體機器。
HP 的解法：Tamper Lock 透過機械感測器偵測攻擊者打開機蓋，並提供可配置的防護機制（警告、立即鎖機、清除 TPM 等），有效防範對個人電腦內部的實體攻擊。
怎麼啟用

預設啟用但可在 BIOS 關閉。Policy 可設「警告但不鎖機」或「立即鎖機」。

何時觸發

底蓋被拆開（含為了升級 SSD、清灰塵）。

事件後處理

螢幕顯示警告 → 需 BIOS 管理員密碼或 Sure Admin 授權才能解除。建議拆機前先進 BIOS 暫關 Tamper Lock。

⚠️ 門市拆機升級 SSD 重要提醒（Tamper Lock 觸發）

這 3 款 G1i 雖然 RAM 焊死無法升級，但 SSD 是 M.2 插槽可換。客戶若要從 512GB 升 1TB / 2TB，店內拆機時會直接觸發 Tamper Lock，可能導致：
  • 下次開機螢幕跳警告，需 BIOS 密碼解除
  • 若客戶設過 BIOS 密碼 → 必須由客戶輸入才能繼續
  • 若已設 Sure Admin → 必須由管理員手機授權
建議 SOP（按順序）：
  1. 升級前讓客戶簽授權單，註明會拆機
  2. 開機按 F10 進 BIOS → Security → Tamper Lock → 暫時 Disable
  3. 關機拆機升級 SSD
  4. 裝回後開機進 BIOS → 把 Tamper Lock 改回 Enable（或維持 Warning Only）
  5. 重要：若客戶已設 BIOS 密碼或交付給 IT 管，請先請客戶或 IT 操作 Step 2，否則拆機後會卡住
結論：可以升級，但要走流程。不建議「先拆再說」，會讓客戶取機時看到警告畫面誤以為被駭。
版本差異提醒：這三款 G1i 預裝的是 HP Wolf Security for Business（標配版）。 市面上還有 HP Wolf Pro Security Edition（進階版，含 Sure Click Pro / Sure Sense Pro 功能更全），需另購授權。客戶問「是不是 Pro 版」誠實回答「標配是 Business 版，Pro 可加購」。

HP AI Companion — 真相揭露

是「獨立 App」不是 Copilot 內建
先解開誤解

HP AI Companion ≠ Microsoft Copilot · 兩個不同的 App，互補使用

很多人誤以為按 Copilot 鍵會開到 HP AI Companion，其實不是：
  • Copilot 鍵：開啟「Microsoft Copilot」（雲端為主，跟 Office 整合）
  • HP AI Companion：是 HP 預裝的獨立 App，需從開始選單或工具列開啟
兩者並存、互補：Copilot 強在 Office / 雲端整合；HP AI Companion 強在本機檔案分析、隱私保護、離線可用。
背後的 AI 模型

用微軟 Phi-3.5（本機）+ GPT-4o（必要時上雲）混合架構

根據 HP 官方文件，AI Companion 採「本機優先」策略：
  • 本機運算：用 Microsoft Phi-3.5 模型（3.8B 參數，跑在 NPU 上），處理文件摘要、本機問答
  • 雲端輔助：複雜任務或需要最新資訊時，使用者明確同意後切到 GPT-4o
所以隱私敏感的問題（合約、客戶資料、會議紀錄）都在本機完成；只有需要查最新新聞、深度推論時才上雲。
40-47 TOPS 真的能跑這些功能嗎？

各功能在 Phi-3.5 / 47 TOPS NPU 上的實際表現

功能 實際表現 速度 備註
文件 / PDF 本機摘要 精準可用 10–20 秒 / 5 頁 Phi-3.5 強項，10 頁內表現很好
會議紀錄整理 精準可用 15–30 秒 適合短會議；超長錄音建議分段處理
即時翻譯 精準可用 近即時 由 Windows Live Captions 配合 NPU 執行
「我筆電為什麼卡」診斷 精準可用 立即 偏向系統規則查詢，不靠 LLM 推論
跨 App 搜尋（撈 Outlook 信） 部分可用 5–15 秒 需搭配 Windows Recall（Copilot+ 功能）；AI Companion 本身不直接撈 Outlook
長文書寫 / Email 草稿 堪用 慢，10–30 秒/段 品質約 ChatGPT 3.5，比不上 GPT-4 / Claude 3.5
程式碼撰寫 勉強 慢 3.8B 模型不擅長，建議用雲端 Copilot
專業學術問答 / 複雜推論 不建議 — 本機模型太小，建議切到 GPT-4o 雲端模式
誠實的銷售話術：不要對客戶說「能跑 ChatGPT 4」，要說「本機跑 Phi-3.5，能力大約 ChatGPT 3.5 等級，但資料完全不外洩，需要更強時可切到雲端 GPT-4o 模式」。專業問答、寫程式建議用雲端版本。

HP Connect — 企業 IT 集中管理 BIOS

適用 50 人以上 · 已用 Microsoft Intune 的客戶
一句話定義

IT 管理員用雲端後台遠端統一管理全公司 HP 筆電的 BIOS 設定

一般公司買一批筆電，IT 要一台一台進 BIOS 設定（禁止開機 USB、設密碼、關閉閒置埠口）。 一台 10 分鐘，50 台就一整天。HP Connect（網址 connect.admin.hp.com）跟企業常用的 Microsoft Intune 串接 —— 寫好一份「公司 BIOS 政策」後，所有 HP 筆電開機連網就自動套用。
實際做什麼客戶實際好處
遠端統一設 BIOS 密碼、改開機順序、關 USB 埠IT 從「每台 10 分鐘」降到「政策寫一次全部套用」
BIOS 韌體升級雲端推播全公司新人報到發筆電當天可用
偵測 BIOS 設定被動過、通報管理員員工不能私下改 BIOS 繞過管制
搭配 Sure Admin 手機授權臨時進 BIOS資安稽核時可拉「全公司筆電 BIOS 狀態報表」一次交代
裝置端不用裝軟體，靠 Intune 原生機制零軟體部署 → IT 維運成本下降
適用門檻提醒：HP Connect 的價值在「台數多」時才明顯。家用、小店面、5 人內公司用不到。建議在針對 50 人以上、已用 Microsoft 365 Business 方案 的 B2B 客戶銷售時主動提。
